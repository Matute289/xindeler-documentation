# 004 — i18n ES + EN

**Estado:** `[ ]` Pendiente  
**Prioridad:** Media  
**Sprint:** 1  
**Nota:** El contenido inicial se escribe en ES. EN se agrega después.

## Qué hacer

Docusaurus tiene i18n nativo. Para agregar ES como segundo idioma:

```bash
npm run write-translations -- --locale es
```

Esto genera `i18n/es/` con los strings de UI traducibles.

### Estrategia
- Escribir todo el contenido en **ES primero**
- Traducir a EN una vez que el contenido ES está maduro
- Usar `docusaurus.config.ts` que ya tiene `locales: ['en', 'es']` configurado

### Para traducir un doc
Copiar el archivo de `docs/` a `i18n/es/docusaurus-plugin-content-docs/current/` manteniendo la misma ruta.
